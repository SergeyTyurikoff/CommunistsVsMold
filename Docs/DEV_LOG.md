## Как это работает (инструкция для Claude)

- **«Сделано» = коммиты.** Итерация ≈ один коммит (`Unity: …`). Лог не дублируем —
  `git -C C:\prog\brainiac\unity log --oneline`. Сообщения коммитов по-русски.
- **«Следующее» = бэклог ниже.** Мелко: итерация ≈ ~3 ч будни / ~8 ч выходные, день-два, = коммит.
- **Направление согласуем.** Перед декомпозицией/`0` — показать владельцу «что предлагаю дальше»,
  получить подтверждение (особенно по сюжету/арту — это его авторские решения).
- **🐞 Баги — отдельный тип итерации.** Ошибки накапливаются → периодически планировать
  итерацию «собрать и починить накопленные баги» (источник: рантайм-QA `unity-qa`, TODO в коде,
  жалобы владельца). Не откладывать бесконечно.
- **Связь с деревом «Основа».** `декомп` (и старт `0`) берёт верхние пункты «Следующих
  итераций» → подзадачами под «Сделать Коммунисты» (проект «Стать геймдизайнером»,
  `POST /api/planner/tasks projectId=pr_z`), цель `deps`→веха, снять `decompose`, разложить даты.
- **Цикл:** взять пункт → (опц.) в дерево → сделать в Unity (`unity-coder`/сцена/QA по правилам
  `кп`/`турбо` в корневом CLAUDE.md) → **коммит** → **грумить этот файл** (убрать сделанное, дописать новое).
