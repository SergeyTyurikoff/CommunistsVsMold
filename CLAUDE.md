# CLAUDE.md — репозиторий разработки Unity-игры «Коммунисты против… Плесени»

Это git-репозиторий **разработки игры** (Unity-проект — подпапка `CommunistsVsMold\`).
Часть воркспейса `C:\prog\brainiac`, но обычно открывается **отдельно** (Unity требует
короткий ASCII-путь). Поэтому ключевые правила продублированы здесь.

## Главное: веди разработку через дев-лог итераций

Источник процесса — **`Docs\DEV_LOG.md`** (там конвенция, текущее состояние, бэклог,
хвосты/риски). Кратко:

- **«Сделано» = git-история** (`git -C C:\prog\brainiac\unity log --oneline`) — НЕ дублировать в доку.
- **«Следующее» = раздел «Следующие итерации»** в DEV_LOG. Одна итерация ≈ один коммит
  (~3 ч в будни / ~8 ч на выходных), сообщения коммитов **по-русски**.
- **Направление (что делать дальше) — СОГЛАСОВЫВАТЬ с владельцем**, не выбирать молча
  (особенно сюжет/диалоги/катсцены/замену локаций/персонажей/спрайтов — это его авторские решения).
- **🐞 Баги — отдельный тип итерации:** накапливаются → периодически собрать (рантайм-QA + TODO в
  коде) и починить отдельной итерацией. Не откладывать бесконечно.
- **После коммита итерации — груми DEV_LOG:** убери сделанный пункт (он уже в git), допиши новое.

Полная инструкция, бэклог и список хвостов — в `Docs\DEV_LOG.md`. Сверяйся с ним В НАЧАЛЕ сессии.

## Связь с планировщиком «Основа» (дерево задач)

Прямой автоматической связи НЕТ (это другой репозиторий/зона — `99_Программы\Интерфейсы\Основа`).
Синхронизация **DEV_LOG → дерево** делается **вручную, командой `декомп` или стартом `0`** в чате с
поднятой «Основой»: они берут верхние «Следующие итерации» и заводят их подзадачами под задачу
**«Сделать Коммунисты»** (проект «Стать геймдизайнером»). То есть разделение:

- **здесь (этот репо/чат)** — пилим игру в Unity + ведём `DEV_LOG.md`;
- **в brainiac-чате с «Основой»** — синхроним `DEV_LOG` → дерево (`декомп`/`0`).

Это разделение by design: дев-чат не обязан трогать планировщик, а планировщик читает дев-лог.

## Технические правила (Unity-агенты, QA)

- Игровой код — в `Assembly-CSharp` (asmdef нет): `run_tests` недоступен без рефактора → QA через
  `execute_code` (Unity MCP), игрока на время теста замораживать Kinematic.
- Длинные C#-блоки в `execute_code` дробить (длинный payload срывает MCP-вызов в текст).
- Unity ломается на длинном кириллическом пути (MAX_PATH / DirectoryNotFoundException) — держать ASCII.
- Полные правила геймдев-цикла, `турбо`-фан-аут по `unity-coder` и подводные камни Unity MCP —
  в корневом `C:\prog\brainiac\CLAUDE.md` (если открыт весь воркспейс).
